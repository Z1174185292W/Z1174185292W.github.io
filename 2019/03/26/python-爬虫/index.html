–limit-rate=200k 以指定的速度下载目标文件 wget –limit-rate=200k https://ks-xpc19.xpccdn.com/f6061729-e55b-4ac1-8228-e19c1801544e.mp4

tail -f wget-log
-U 设置User-Agent wget -U “Windows IE 10.0” -bc –limit-rate=200k https://ks-xpc19.xpccdn.com/f6061729-e55b-4ac1-8228-e19c1801544e.mp4

-r 递归下载所有网页中所有的链接

四、中间件的调用规则

process_request

在request对象传往downloader的过程中和将下载结果返回给engine过程中调用。当返回不同类型的值得时候，行为也不一样：

返回值 行为
None 一切正常，继续执行其他的中间件链
Response 停止调用其他process_request和process_exception函数，也不再继续下载该请求，然后走调用process_response的流程
Request 不再继续调用其他process_request函数，交由调度器重新安排下载
IgnoreRequest process_exception函数会被调用，如果没有此方法，则request.errback会被调用，如果errback也没有，则此异常会被忽略，甚至没有日志。

process_response

在将下载结果返回给engine过程中调用。

返回值 行为
Response 继续调用其他中间件的process_response
Request 不再继续调用其他process_request函数，交由调度器重新安排下载
IgnoreRequest request.errback会被调用，如果errback也没有，则此异常会被忽略，甚至没有日志。

process_exception

在下载过程中出现异常，或者在process_request中抛出IgnoreResponse异常的时候调用。

返回值 行为
Response 开始中间件链的process_response处理流程
Request 不再继续调用其他process_request函数，交由调度器重新安排下载
None 继续调用其他中间件里的process_exception函数

from_crawler(cls, crawler)

如果存在该函数，则调用该函数创建中间件的实例。如果要写这个函数，一定要返回一个中间件的对象。
